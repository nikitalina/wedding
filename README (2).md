# 💍 Свадебное приглашение — сайт-одностраничник

Лендинг-приглашение на свадьбу с RSVP-формой и интеграцией с Google Sheets.  
Размещается на **GitLab Pages** — бесплатно, с HTTPS и поддержкой кастомного домена.

## Структура файлов

```
├── index.html              ← Весь сайт (HTML + CSS + JS в одном файле)
├── google-apps-script.js   ← Скрипт для Google Apps Script (бэкенд формы)
├── .gitlab-ci.yml          ← Конфиг CI/CD для деплоя на GitLab Pages
└── README.md               ← Этот файл
```

---

## Быстрый старт

### 1. Настройте Google Sheets (бэкенд для формы)

Форма RSVP отправляет данные в Google Таблицу через Google Apps Script.  
Бэкенд-сервер не нужен — всё работает на стороне Google бесплатно.

1. Создайте таблицу: [sheets.new](https://sheets.new)
2. В строку 1 (заголовки) добавьте:

   | A | B | C | D |
   |---|---|---|---|
   | Дата | ФИО | Контакт | Присутствие |

3. Откройте **Extensions (Расширения) → Apps Script**
4. Удалите код по умолчанию и вставьте содержимое файла `google-apps-script.js`
5. Нажмите **▶ Run** рядом с функцией `initialSetup`, подтвердите разрешения Google
6. Перейдите в **Deploy → New deployment**
   - **Select type:** Web app
   - **Execute as:** Me
   - **Who has access:** Anyone
7. Нажмите **Deploy** и скопируйте URL (вида `https://script.google.com/macros/s/XXXX.../exec`)

> ⚠️ Если позже измените Apps Script — создавайте **новый** деплоймент,  
> иначе изменения не подхватятся (это особенность Google Apps Script).

---

### 2. Настройте сайт

Откройте `index.html` в любом редакторе (VS Code, Sublime, даже Блокнот).  
Найдите поиском `TODO` — все места для редактирования помечены комментариями.

Что заполнить:

- **Имена** жениха и невесты — в секции hero и footer
- **Дата** свадьбы
- **Фото** — замените блок `<div class="photo-placeholder">` на `<img src="photo.jpg" alt="Фото">`
- **Историю** пары (секция «Наша история»)
- **Детали** — место, время, адреса, дресс-код (карточки)
- **Программу дня** — таймлайн с расписанием
- **`GOOGLE_SCRIPT_URL`** — вставьте URL из шага 1.7
- **OG-теги** — для красивого превью в мессенджерах (WhatsApp, Telegram)
- **Ссылку** на себя в footer

> 💡 Совет: фото оптимизируйте через [tinypng.com](https://tinypng.com) до ≤300 КБ.  
> Рекомендуемый размер: 800×1067px (соотношение 3:4, портретное).

### 3. Разместите на GitLab Pages

#### 3.1. Создайте аккаунт

Если ещё нет — зарегистрируйтесь на [gitlab.com](https://gitlab.com).  
Бесплатного тарифа хватит с запасом (10 ГБ хранилища, 100 ГБ трафика/мес).

#### 3.2. Создайте новый проект

1. Нажмите **New project → Create blank project**
2. Заполните:
   - **Project name:** например, `wedding` или `svadba`
   - **Visibility Level:** Public (чтобы Pages были доступны всем)
   - ☐ Уберите галочку «Initialize repository with a README» (мы загрузим свой)
3. Нажмите **Create project**

#### 3.3. Создайте файл `.gitlab-ci.yml`

Этот файл говорит GitLab, что и как деплоить.  
Положите его **в корень проекта** рядом с `index.html`.

Содержимое `.gitlab-ci.yml`:

```yaml
# Конфиг для деплоя статического сайта на GitLab Pages
# Документация: https://docs.gitlab.com/ee/user/project/pages/

image: alpine:latest

pages:
  stage: deploy
  script:
    - mkdir .public
    - cp -r * .public
    - mv .public public
  artifacts:
    paths:
      - public
  rules:
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
```

> **Что тут происходит:** GitLab CI запускает контейнер Alpine Linux,  
> копирует все файлы в папку `public` и публикует её как сайт.  
> Происходит автоматически при каждом push в ветку main.

#### 3.4. Загрузите файлы

**Вариант A — через веб-интерфейс (самый простой):**

1. В репозитории нажмите **+ → Upload file**
2. Загрузите: `index.html`, `.gitlab-ci.yml`, ваше фото (если есть)
3. Напишите Commit message, например: `Добавить свадебный сайт`
4. Нажмите **Upload file**

> ⚠️ Файл `.gitlab-ci.yml` начинается с точки — убедитесь, что ваш файловый  
> менеджер показывает скрытые файлы, иначе вы его не увидите на диске.

**Вариант B — через Git (если хотите работать локально):**

```bash
# Склонируйте пустой репозиторий
git clone https://gitlab.com/YOUR_USERNAME/wedding.git
cd wedding

# Скопируйте файлы сюда (index.html, .gitlab-ci.yml, фото и т.д.)

# Закоммитьте и запушьте
git add .
git commit -m "Свадебное приглашение"
git push origin main
```

> 💡 **Шпаргалка по Git-командам:**  
> `git clone <url>` — скачать репозиторий к себе на компьютер  
> `git add .` — добавить все файлы в папке к коммиту  
> `git commit -m "текст"` — создать «снимок» текущих изменений с описанием  
> `git push` — отправить изменения на GitLab (после этого автоматически запускается деплой)  
> `git pull` — скачать свежие изменения с GitLab к себе

#### 3.5. Дождитесь деплоя

1. Перейдите в **Build → Pipelines** в левом меню репозитория
2. Вы увидите запущенный pipeline — подождите пока значок станет ✅ зелёным
3. Обычно занимает 1–3 минуты

> Если pipeline красный ❌ — нажмите на него, потом на job `pages`,  
> и посмотрите лог ошибки. Чаще всего это опечатка в `.gitlab-ci.yml`  
> (YAML чувствителен к пробелам — используйте пробелы, не табы).

#### 3.6. Откройте сайт

Адрес сайта: **`https://YOUR_USERNAME.gitlab.io/wedding/`**

Также можно найти URL в **Deploy → Pages** в левом меню репозитория.

> 🎉 Если видите свой сайт — поздравляю, он онлайн!  
> Теперь можно скинуть ссылку друзьям для тестирования.

### 4. Подключите кастомный домен (опционально)

Красивый домен типа `anna-i-ivan.ru` вместо `username.gitlab.io/wedding`.

#### 4.1. Купите домен

У любого российского регистратора — так DNS будут стабильно работать из РФ:
- [reg.ru](https://reg.ru) — от ~200₽/год за .ru
- [beget.com](https://beget.com) — от ~200₽/год
- [nic.ru](https://nic.ru) — от ~200₽/год

#### 4.2. Настройте GitLab

1. В репозитории: **Deploy → Pages**
2. Нажмите **New domain**
3. Введите ваш домен (например, `anna-i-ivan.ru`)
4. GitLab покажет **верификационный TXT-код** — запишите его

#### 4.3. Настройте DNS у регистратора

В панели управления доменом добавьте DNS-записи:

| Тип | Хост / Имя | Значение |
|-----|-----------|----------|
| **A** | `@` | `35.185.44.232` |
| **TXT** | `_gitlab-pages-verification-code` | Код из шага 4.2 |
| **CNAME** | `www` | `YOUR_USERNAME.gitlab.io` |

> ⏳ DNS-записи обновляются от 10 минут до 24 часов (зависит от регистратора).  
> Проверить распространение можно на [dnschecker.org](https://dnschecker.org).

#### 4.4. Включите HTTPS

После верификации домена GitLab Pages автоматически выпускает  
бесплатный SSL-сертификат от Let's Encrypt.

В **Deploy → Pages** убедитесь, что опция **Force HTTPS** включена.

## Как вносить изменения после деплоя

Любые правки автоматически деплоятся после push.

**Через веб-интерфейс (не нужен Git):**
1. Откройте файл в репозитории → нажмите **Edit → Edit single file**
2. Внесите изменения → **Commit changes**
3. Через 1–3 минуты изменения появятся на сайте

**Через Git локально:**
```bash
# Откройте папку проекта и внесите правки в файлы, затем:
git add .
git commit -m "Обновить время банкета"
git push
```

## Что где редактировать

Все плейсхолдеры помечены в коде комментариями `<!-- TODO: ... -->`.  
Поиск по `TODO` в любом редакторе покажет все места.

| Что | Где в коде |
|-----|-----------|
| Имена пары | Секции `hero`, `footer` |
| Дата свадьбы | Секция `hero`, `footer`, OG-теги |
| Фото | Секция `photo-section` — заменить placeholder на `<img>` |
| История | Секция «Наша история» |
| Место / время / дресс-код | Карточки в секции «Детали» |
| Расписание | Таймлайн в секции «Программа дня» |
| Google Script URL | Переменная `GOOGLE_SCRIPT_URL` в `<script>` |
| Превью в мессенджерах | OG-теги в `<head>` |
| Ссылка на автора | Footer, блок `.footer-credit` |

## Решение проблем

| Проблема | Решение |
|----------|---------|
| Pipeline красный ❌ | Проверьте `.gitlab-ci.yml` — пробелы, не табы. Скопируйте из README заново |
| Сайт 404 | Убедитесь, что файл называется `index.html` (не `Index.html`) |
| Форма не отправляется | Проверьте `GOOGLE_SCRIPT_URL` в `index.html` |
| Фото не грузится | Проверьте путь — файл фото должен лежать в том же репозитории |
| Сайт не обновился | Подождите 2–3 мин после push, проверьте pipeline в Build → Pipelines |
| Домен не работает | Проверьте DNS на [dnschecker.org](https://dnschecker.org), подождите до 24ч |
