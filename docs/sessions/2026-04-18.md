### 1.2. Блок «Просьба» (про цветы) — варианты предложены, финальная формулировка не выбрана

**Где:** [index.html:126-132](../../index.html#L126-L132)
**Текущий текст:**
> Просьба не дарить цветы. Вместо цветов будем рады вкладу в наш семейный фонд.

**Проблема:** невесте не нравится формулировка — звучит слишком директивно и бюрократично («семейный фонд»).

**Смысл, который нужен:** не хотели бы получать цветы (они увянут); если гости захотят сделать подарок — лучше конвертом, чтобы не тратились впустую.

**Предложенные варианты (ожидают выбора):**
1. *Нежно и с заботой о гостях:* «Мы будем благодарны, если вы не станете тратиться на цветы — к вечеру они, к сожалению, увянут. Ваше присутствие для нас — лучший подарок, а если захочется добавить что-то ещё, мы будем рады конверту.»
2. *Сдержанно и изящно:* «Пожалуйста, не беспокойтесь о цветах — пусть они останутся в саду. Если же вам хочется нас порадовать, мы будем благодарны за конверт.»
3. *Коротко и тепло:* «Самый дорогой подарок для нас — видеть вас рядом в этот день. Если захочется дополнить его — мы будем признательны конверту вместо букета.»
4. *С лёгкой ноткой:* «Букеты, увы, недолговечны — а мы хотим запомнить этот день надолго. Если хочется нас порадовать, мы будем рады конверту.»

**Решение:** отложено — ждём выбор.

## 2. Настройка Claude Code

### 2.1. Команды для просмотра использования
Разобрали, какими слеш-командами смотреть расход и состояние сессии:
- `/cost` — токены и стоимость текущей сессии
- `/context` — что занимает контекстное окно (%)
- `/status` — модель, версия, путь
- `/model` — текущая/переключение модели
- `/help` — полный список команд
- Точные лимиты подписки — на странице Usage в Claude Console

### 2.2. Настроен кастомный статус-лайн

**Цель:** видеть в статус-баре модель, папку, git-ветку и индикатор контекста.

**Созданные/изменённые файлы:**
- `C:\Users\trine\.claude\statusline.sh` — bash-скрипт, парсит JSON от Claude Code через `grep`/`sed` (jq и node недоступны в среде), достаёт модель и cwd, получает ветку через `git branch --show-current`.
- `C:\Users\trine\.claude\settings.json` — добавлен блок:
  ```json
  "statusLine": {
    "type": "command",
    "command": "bash /c/Users/trine/.claude/statusline.sh"
  }
  ```

**Формат вывода:**
```
[Opus] WeddingProjject (main) | ctx <200k
```

**Оговорка:** точный % контекста Claude Code скрипту не отдаёт — только булев флаг `exceeds_200k_tokens`. Для точного значения — `/context`.
